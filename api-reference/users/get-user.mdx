## Overview

Retrieve detailed information about a specific user. This endpoint requires appropriate permissions based on your relationship to the user.

## Authentication

This endpoint requires authentication. Include your API key or access token in the request header.

## Path Parameters

<ParamField path="user_id" type="string" required>
  The unique identifier of the user you want to retrieve. Use "me" to get the current user's profile.
</ParamField>

## Response

<ResponseField name="id" type="string" required>
  Unique user identifier
</ResponseField>

<ResponseField name="email" type="string" required>
  User's email address
</ResponseField>

<ResponseField name="name" type="string" required>
  User's display name
</ResponseField>

<ResponseField name="avatar_url" type="string">
  URL to the user's profile picture
</ResponseField>

<ResponseField name="status" type="string" required>
  User account status: `active`, `inactive`, or `suspended`
</ResponseField>

<ResponseField name="role" type="string" required>
  User's role: `user`, `admin`, or `moderator`
</ResponseField>

<ResponseField name="metadata" type="object">
  Custom metadata associated with the user
</ResponseField>

<ResponseField name="created_at" type="string" required>
  ISO 8601 timestamp when the user was created
</ResponseField>

<ResponseField name="updated_at" type="string" required>
  ISO 8601 timestamp when the user was last updated
</ResponseField>

<ResponseField name="last_login_at" type="string">
  ISO 8601 timestamp of the user's last login
</ResponseField>

## Permissions

Access to user information depends on your authentication level and relationship to the user:

### Own Profile
- **Any authenticated user** can access their own profile using `/users/me`
- Full access to all profile information including private metadata

### Other Users
- **Regular users** can access basic information (name, avatar, public metadata) of other users
- **Admins** can access full information for any user
- **Moderators** can access extended information for users in their domain

### Field Visibility

| Field | Own Profile | Other Users (Basic) | Other Users (Admin) |
|-------|-------------|-------------------|-------------------|
| `id` | ✅ | ✅ | ✅ |

## Common Use Cases

### Profile Page
Display user information on a profile page:

```javascript
// Get current user for profile page
const currentUser = await getUser('me');
document.getElementById('username').textContent = currentUser.name;
document.getElementById('email').textContent = currentUser.email;
```

### User Directory
List basic information about multiple users:

```javascript
// Get user info for directory listing
const users = await Promise.all(
  userIds.map(id => getUser(id))
);
```

### Admin Dashboard
Access detailed user information for administration:

```javascript
// Admin view with full details
const userData = await getUser(selectedUserId);
console.log('Last login:', userData.last_login_at);
console.log('Account status:', userData.status);
```

## Error Handling

<Accordion title="Common Error Scenarios">

**User Not Found (404)**
- The specified user ID doesn't exist
- The user has been deleted
- You don't have permission to see the user

**Insufficient Permissions (403)**
- Trying to access another user's private information
- API key doesn't have required scopes
- User is suspended or inactive

**Invalid User ID (400)**
- User ID format is invalid
- Using reserved keywords incorrectly

</Accordion>
